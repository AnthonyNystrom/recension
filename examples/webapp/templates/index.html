recension live demo

Watch the library optimize a text artifact against held-out evidence. Each run is the real propose/test/accept loop against the deterministic offline mock: no API key, fully reproducible.

The usual way to improve a prompt is to edit it, eyeball a few outputs, and ship. Nothing measures whether the edit helped, nothing records why it was made, and nothing stops it from overfitting to the cases you happened to inspect. recension treats a prompt as a production artifact: it accepts an edit only when it beats the incumbent on held-out data, checks the gain wasn't bought by memorizing that data, and records every decision so a reviewer can replay it.

Propose candidate edits from diagnosed failures → collate by scoring each on a held-out split → commit only what measurably beats the incumbent, with provenance.

  • Classification & extraction: support-ticket triage, invoice fields, moderation
  • RAG context templates: tune how chunks are assembled, model held fixed
  • Agent & skill instructions: longer files judged by a rubric
  • Governance & audit: a replayable record of why every prompt changed

Pick a scenario and run it. Each is a small, real instance of the use cases above.
